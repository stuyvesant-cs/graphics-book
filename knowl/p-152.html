The semantic analyzer takes the syntax tree and create a list of operations in the order that they need to be performed so that the program can work. Given the syntax tree above, the semantic analyzer might make an operations list like the following:
        MULTIPLY 5, 8
        ASSIGN x PREVIOUS
        printf "%d", x
The ASSIGN entry would have some reference to the previous entry. Things like x and printf would refer to the symbol table, since they are identifiers. The operations list is designed such that it can then be directly translated into processor operations, ready to be translated into binary code.
